标题：技术和技能：工具还是终极目标？

技术和技能终究只是工具，其价值在于综合达到特定目标，无论是实现业务的稳定性还是快速抢占市场。稳定性和敏捷性之间的平衡始终存在矛盾，但好的技术应该能够在其中找到平衡。

如果您想在前端领域成为专家，React 是一个很好的选择。如果您在小公司工作或者想快速上线业务以获取收益，Vue 和甚至 jQuery 可能是更合适的工具。

随着人工智能的发展，AI 生成式框架可能会崭露头角，因为它们更灵活，学习曲线更低。

2019年，前端市场中，React 大约占了八成，Angular 占有约一成半，Vue 大约占了半成左右。这些数据主要来自猎头电话、邮件、LinkedIn 邀请以及与同行和猎头的交流。因为我在这些方面的信息比较全面。在这个市场上，人们普遍认为 Angular 复杂、过时且学习困难，React 是主流选择，而 Vue 则被认为新颖且适合初学者。

同样，在国外的招聘市场，React 早已占据主导地位，几乎每个人都懂 React，跳槽的机会几乎都与 React 有关。

总之，React 更灵活，有更多机会自己开发组件，这将带来更多的产出，最终可能会带来晋升和加薪。

目前的趋势看来，需要一套同时具备 React 和 Vue 特点的框架：既可以像 Vue 一样使用模板化，又可以像 React 一样实现数据分离。Vue 的全家桶框架省去了很多心智负担，而 React 可能需要更多的时间来满足项目需求。

不得不说，使用 Vue 加上组件库开发自制博客非常快速，而且写文章时只需复制模板文件，然后将自己的 Markdown 文章粘贴进去即可。

另一个方面是维护性，Vue 完全由社区支持，而 React 和 Angular 由公司开发和维护。对于企业来说，某些问题可以通过资金来解决，尤其是在金融系统等重要领域，时间至关重要，因此有时候花钱来找人维修是必要的。Vue 这种社区驱动的软件无法提供及时的技术支持。

总结：

对于小公司来说，生存至关重要。因此，首要目标是高效验证业务的可行性和快速迭代。在这种情况下，Vue 这样有模板语法约束的工业级产品更适合小公司。通常的配置是：大公司的资深领导解决架构问题和技术难题，然后雇佣初中级前端开发者，大家都使用 Vue 模板语法，即使是相对不熟练的开发者，代码质量也是可控的。

大公司不仅在用户量上有巨大差异，业务体量也更大。因此，大公司可以从改进开发流程、提高开发性能和效率中获得数量级的收益增长。React 更接近纯粹的 JavaScript，团队水平高的情况下，开发效率也非常高，而且能够快速构建各种组件。前辈们可以使用 React 快速搭建项目，验证业务可行性，然后晋升加薪离职。后继者可以优化（重构）前人的代码，再次获得收益，然后也晋升加薪离职。这一连串的传承可以持续下去，因此 React 可以成为一个职业生涯的长期选择。

因此，如果想进入大公司，最好充分掌握 React。
